# Batch offset=0 — implementation note

## Command
```
python3 scripts/ingest-batch.py --offset 0 --limit 100
```

But actually this batch was done by a subagent before the script existed. The subagent worked directly with file writes.

## What was done
- Fetched `GET /api/v1/courses/?offered_by=ocw&limit=100&offset=0`
- Created 100 course pages in `wiki/courses/`
- Created 168 instructor pages in `wiki/instructors/`
- Updated all 37 department pages with course links
- Updated 80 topic pages with course links (30 topics had no courses in this batch)
- Saved raw JSON to `raw/api/courses-page-000.json`
- Updated `_checkpoint.json`: batches_done=1, last_offset=100, courses_in_wiki=104

## Anomalies
- The DRF API returns HTML-wrapped JSON, not raw JSON. The extraction required parsing HTML to find the embedded JSON blob.
- Some topic pages (30 of 110) had no courses in this batch — expected since topics are spread across all 26 pages.
